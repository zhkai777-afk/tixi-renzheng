# 页面统一风格规范

## 风格基准

本项目所有页面优先对齐 `二次评定-详情.html` 的后台业务系统风格。整体气质应是清晰、克制、专业、高信息密度，不使用营销页式大标题、装饰图形或大面积渐变。

## 布局

- 页面使用顶部固定信息栏 + 主体三栏布局。
- 顶部栏高度约 `64px`，白底，底部 `1px #e7e7e7` 分割线。
- 主体背景为 `#f5f5f5`。
- 左侧导航栏宽度约 `320px`，白底，右边框。
- 中间内容区自适应，占主要空间，内部滚动。
- 右侧控制台宽度约 `360-380px`，白底，左边框。
- 页面优先保持桌面后台宽屏体验，允许横向最小宽度约 `1848px`。

## 色彩

体系主色：
- Q：`#0052D9`
- E：`#00A870`
- S：`#FF7D00`
- F：`#7B4DFE`

体系标签统一使用弱化色，不使用高饱和彩底白字：
- Q：背景 `#E8F1FF`，边框 `#B7D3FF`，文字 `#0052D9`
- E：背景 `#E6F7F1`，边框 `#B8E8D7`，文字 `#00875A`
- S：背景 `#FFF3E5`，边框 `#FFD5A8`，文字 `#C46000`
- F：背景 `#F1EDFF`，边框 `#D6CAFF`，文字 `#6339D8`
- 认证领域筛选生效时，未选中的体系标签统一灰化：背景 `#F2F3F5`，边框 `#DCDDE1`，文字 `#9CA3AF`。

状态颜色：
- 成功/通过：`#00A870`
- 驳回/缺失/错误：`#E34D59`
- 待处理/警告：`#ED7B2F` 或 `#FF7D00`
- 普通文字：`rgba(0,0,0,0.9)`
- 次级文字：`rgba(0,0,0,0.6)`
- 辅助文字：`rgba(0,0,0,0.4)`
- 边框：`#e7e7e7`
- 表格浅底：`#fafafa`

## 组件

按钮：
- 高度约 `32-36px`。
- 圆角 `4px`。
- 主按钮使用 Q 蓝色背景。
- 成功动作可用绿色，危险动作用红色，次要动作白底描边。
- 禁用态降低透明度并禁止点击。

标签：
- 体系徽标使用单字母 `Q/E/S/F`，浅底深字、细边框、小圆角。
- 状态标签使用浅色背景 + 深色文字。
- 不使用 emoji 作为关键业务图标。

面板：
- 白底、细边框、`8px` 圆角。
- 标题区常用 `#fafafa` 背景和底部分割线。
- 面板之间通过 `16-20px` 间距区分。

表格：
- 表头 `#fafafa`，13px，字重 600。
- 单元格 13px，纵向居中，底部分割线。
- 行 hover 使用 `#fafafa`。
- 需关注项可使用浅黄底 `#FFFBE6`。

中间栏材料明细表：
- 中间栏展示具体文件时统一使用“材料明细表”面板，白底、`1px #dfe7f2` 边框、`8px` 圆角。
- 面板标题区高度约 `56px`，左侧标题为 15px/600，右侧显示“当前显示 N 个具体文件，来自 M 项材料”等统计，文字 12px、灰蓝色。
- 表格列固定为“具体文件名称 / 适用体系 / 操作”，不在主表列中单独展示类型、带入环节、上传时间或评定状态；这些信息如需展示，应作为文件名下方辅助信息或详情弹窗内容。
- “具体文件名称”列第一行显示文件名，14px、600；第二行显示“所属材料：xxx”，12px、灰蓝色。
- “适用体系”列使用 Q/E/S/F 单字母浅色徽标，保持与全站体系色一致。
- 初次评定、二次评定和专家评审不对单个具体文件给出评定状态；如果文件存在问题，应登记到所属材料清单项的问题清单中。
- “操作”列使用描边按钮，常规操作为“预览”，按钮高度约 `34px`、圆角 `6px`，不使用醒目的实心主按钮。
- 表格行高约 `78px`，底部分割线 `#edf1f5`；可用极浅灰蓝色间隔行辅助扫描，但不得使用高饱和色块。

左侧材料导航：
- 顶部包含标题、统计、筛选、搜索。
- 材料分组支持展开/收起。
- 当前选中项使用浅蓝底和蓝色左边框。
- 缺失材料使用红/橙状态点或标签提示。
- 材料清单项必须对齐 `归档材料提交-详情.html` 的材料树样式：条目内边距 `12px 16px`，左侧 `3px` 选中边框，底部分割线 `#f5f5f5`，hover 使用 `#fafafa`。
- 分组标题使用 `#f7f8fa` 背景和 `#eef0f3` 底部分割线，左侧显示分组名和项数，右侧显示文件数或同级统计。
- 单项材料第一行只显示材料名称；体系徽标放在材料名称下方单独一行，允许多行换行，以兼容 6 个或 8 个体系。
- 第二行使用材料分类/适用说明 + 右侧胶囊标签组合；只有必交材料显示 `必交` 标签，非必交材料不显示标签也不占位。
- 体系徽标在左侧材料树中使用单字母浅底样式，尺寸约 `24px` 宽、`18px` 行高、`4px` 圆角，允许在窄栏内换行。
- “全部材料”入口也应使用同一材料项结构，标题、体系徽标、统计信息保持与普通材料项一致，不单独设计成卡片或列表外样式。
- 专家评审页的左侧材料清单必须按专家授权体系过滤；例如 Q 体系专家只展示包含 Q 标签的材料，完全不包含 Q 的 E/S/F 专属材料不在材料树中展示，统计项数和文件数也按过滤后的材料范围计算。

右侧控制台：
- 用于展示完整性检查、任务状态、提交条件、提示信息。
- 关键统计使用数字突出。
- 不满足提交条件时显示明确缺失原因。

## 交互

- 筛选和搜索应实时更新左侧材料树和中间内容。
- 上传、删除、提交等动作必须有即时反馈或确认弹窗。
- 删除材料需要二次确认。
- 提交前需要校验所有体系必备材料齐全。
- 弹窗遮罩使用半透明黑色背景，弹窗白底、8px 圆角。

## 左侧材料清单规范

左侧材料清单是本项目归档、评定、专家审阅等页面的统一材料导航组件，后续页面应按以下结构和样式实现。

顶部区域：
- 标题固定为当前业务语义，如“归档材料清单”或“归档材料评审清单”。
- 标题下展示统计：总计项数、已完成/已带入数量、缺失数量。
- 认证领域筛选使用多选标签，默认全部；本批模拟数据默认 `Q/E/S/F`。
- 认证类型筛选使用多选标签。
- 快速筛选包含：`全部`、`仅看缺失`、`已带入`。
- 搜索框用于按材料名称实时模糊筛选。

材料树结构：
- 第一项为“全部材料跨体系视图”，展示全部材料数量、当前必交项数量、当前文件数。
- 后续按 MSF11-20 分类分组：
  - 通用材料
  - 一阶段材料
  - 二阶段材料
- 分组标题显示分组名称、材料项数、文件数。
- 材料项点击后，中间栏切换到该材料的文件清单或空状态。

材料项内容：
- 第一行显示材料名称，允许换行，不与体系标签挤在同一行。
- 体系标签放在材料名称下方，横向排列并允许多行换行，必须能容纳 6 个或 8 个体系标签。
- 体系标签使用本规范的弱化色；当认证领域筛选生效时，未选中的体系标签灰化。
- 底部左侧显示材料标签或分类说明，如“通用”“E/S”“F/H”等。
- 底部右侧显示状态信息：
  - 必交材料显示 `必交` 标签。
  - 非必交材料不显示 `非必交` 标签，也不预留占位。
  - 已有文件显示 `N个文件`。
  - 无文件显示 `未上传`。
- 当前选中项使用浅蓝底和蓝色左边框。
- 缺失材料可使用轻量红色/橙色状态提示，但不得压过材料名称。

快捷筛选与中间栏联动：
- `全部`：左侧展示全部材料；中间栏只展示当前范围内已经带入/上传的具体文件，不展示缺失材料行。
- `仅看缺失`：左侧只展示缺失材料；中间栏为空状态，因为当前范围没有具体文件。
- `已带入`：左侧只展示已有文件的材料；中间栏直接展示这些材料下的具体文件。
- 点击单个材料：中间栏展示该材料文件；如果该材料无文件，显示该材料的补充空状态。

禁用浏览器默认弹窗：
- 页面内操作反馈使用自定义 toast 或自定义 modal。
- 不使用浏览器默认 `alert` / `confirm`。

## 文案

- 归档材料提交页主按钮统一为“提交初次评定”。
- 页面标题、状态、提示文案使用业务语言，避免泛化词。
- 空状态要说明当前材料缺失，并给出补充入口。
